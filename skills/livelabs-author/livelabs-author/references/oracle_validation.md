# Oracle Validation

Use this workflow when the workshop covers Oracle Database, SQL, PL/SQL, Oracle AI Database, ORDS, SQLcl, or Oracle-specific feature behavior.

## Prompt The User

Before you finalize an Oracle-related workshop, ask:

- whether they want to load `oracle-db-skills`,
- whether the workshop needs technical cross-checking for commands, feature names, version-specific behavior, or architectural claims.

Use this exact skill name:

- `oracle-db-skills`

## When To Use Oracle DB Skills

Load `oracle-db-skills` when the workshop includes:

- Oracle SQL or PL/SQL,
- Oracle Database package names,
- Oracle AI Database or vector feature claims,
- SQLcl, ORDS, Multitenant, RAC, or performance topics,
- migration guidance that depends on Oracle behavior,
- statements that could be wrong if the Oracle feature or syntax is misremembered.

## What To Validate

Validate:

- package and feature names,
- SQL syntax and object names,
- setup and privilege requirements,
- version-sensitive behavior,
- product claims that sound specific or testable.

## Delivery Expectation

If `oracle-db-skills` was used, report:

- which Oracle topics you cross-checked,
- whether any workshop steps changed after validation,
- any Oracle-specific gaps that still need SME review.
