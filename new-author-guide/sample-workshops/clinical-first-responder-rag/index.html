Clinical scope

Anchor the demo in psychiatry intake, not general medicine.

  • Capture mood, anxiety, sleep, function, safety, supports, substance context, and psychosis or mania flags.
  • Use DSM-5-informed symptom domains to structure intake without turning the demo into a diagnostic system.
  • Add a CBT-style thought and behavior note so the handoff is actionable for the clinician.

Lab 1 Estimated time: 20 minutes

Provision the Oracle foundation and enable the AI-ready workspace.

The learner should leave this lab with an Oracle workspace that is ready for schema creation, APEX pages, ORDS endpoints, and later prompt wiring.

Lab 2 Estimated time: 30 minutes

Model psychiatric intake, rapid assessments, and the grounded knowledge layer.

This lab turns the workshop from a shell into a behavioral-health system. The user should build tables, load approved guidance, and validate how the vector layer will be used later.

Task 1

Model the intake flow as session, message, quick assessment, and doctor handoff records.

  1. Create a session header table that can store presenting concern, route, and current state.
  2. Store each patient or assistant turn in a message table so the transcript can be replayed and summarized later.
  3. Add a rapid assessment table so short scored items can enrich the handoff without living only in UI state.
  4. Create a clinician handoff table that stores the route, safety flags, CBT-style note, report summary, and PDF export state.
Core data model

Create the psychiatry intake and report tables

These tables keep the intake state, the scored items, and the final handoff in one trusted database layer.

CREATE TABLE intake_session (
  session_id            NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  patient_alias         VARCHAR2(120),
  presenting_concern    VARCHAR2(400),
  route_code            VARCHAR2(40),
  status_code           VARCHAR2(30) DEFAULT 'IN_PROGRESS',
  opened_at             TIMESTAMP DEFAULT SYSTIMESTAMP
);

CREATE TABLE intake_message (
  message_id            NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  session_id            NUMBER NOT NULL REFERENCES intake_session(session_id),
  actor_role            VARCHAR2(20),
  domain_tag            VARCHAR2(40),
  message_text          CLOB,
  created_at            TIMESTAMP DEFAULT SYSTIMESTAMP
);

CREATE TABLE rapid_assessment (
  assessment_id         NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  session_id            NUMBER NOT NULL REFERENCES intake_session(session_id),
  item_key              VARCHAR2(60),
  item_label            VARCHAR2(200),
  item_score            NUMBER(1),
  created_at            TIMESTAMP DEFAULT SYSTIMESTAMP
);

CREATE TABLE clinician_handoff (
  handoff_id            NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  session_id            NUMBER NOT NULL REFERENCES intake_session(session_id),
  route_code            VARCHAR2(40),
  safety_flags          CLOB,
  cbt_note              CLOB,
  report_summary        CLOB,
  pdf_status            VARCHAR2(20) DEFAULT 'NOT_GENERATED',
  generated_at          TIMESTAMP
);

CREATE TABLE guidance_chunks (
  chunk_id              NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  topic                 VARCHAR2(120),
  domain_tag            VARCHAR2(60),
  chunk_text            CLOB,
  citation_label        VARCHAR2(180),
  embedding             VECTOR(1536, FLOAT32)
);

CREATE VECTOR INDEX guidance_chunks_hnsw_idx
ON guidance_chunks (embedding)
ORGANIZATION INMEMORY NEIGHBOR GRAPH
DISTANCE COSINE
WITH TARGET ACCURACY 95;

Package contract

                    
Centralize routing and handoff generation in PL/SQL

                    
Keep the contract small enough to teach and strong enough to make the architecture credible.

                  

                  
                

                
CREATE OR REPLACE PACKAGE psych_first_responder_api AS
  FUNCTION select_route (
    p_presenting_concern   IN CLOB,
    p_function_impact      IN CLOB,
    p_safety_signal        IN CLOB,
    p_assessment_total     IN NUMBER
  ) RETURN VARCHAR2;

  FUNCTION build_handoff (
    p_session_id IN NUMBER
  ) RETURN CLOB;
END;
/

CREATE OR REPLACE PACKAGE BODY psych_first_responder_api AS
  FUNCTION select_route (
    p_presenting_concern   IN CLOB,
    p_function_impact      IN CLOB,
    p_safety_signal        IN CLOB,
    p_assessment_total     IN NUMBER
  ) RETURN VARCHAR2 IS
  BEGIN
    IF REGEXP_LIKE(LOWER(p_safety_signal), 'self-harm|unsafe|command hallucination|not safe alone') THEN
      RETURN 'IMMEDIATE_SAFETY_ESCALATION';
    ELSIF REGEXP_LIKE(LOWER(p_safety_signal || '' '' || p_function_impact), 'three days without sleep|unable to function|severe panic')
       OR NVL(p_assessment_total, 0) >= 6 THEN
      RETURN 'URGENT_PSYCHIATRY_REVIEW';
    ELSE
      RETURN 'ROUTINE_PSYCHIATRY_INTAKE';
    END IF;
  END;
END;
/

              


              

                

                  

                    
ORDS contract

                    
Expose a focused handler for the clinician handoff

                    
One clean endpoint is enough for the demo. The learner can see how the app asks for the report without exposing the package internals.

                  

                  
                

                
BEGIN
  ORDS.DEFINE_MODULE(
    p_module_name    => 'psych',
    p_base_path      => '/psych/',
    p_items_per_page => 0
  );

  ORDS.DEFINE_TEMPLATE(
    p_module_name    => 'psych',
    p_pattern        => 'handoff/:session_id'
  );

  ORDS.DEFINE_HANDLER(
    p_module_name    => 'psych',
    p_pattern        => 'handoff/:session_id',
    p_method         => 'GET',
    p_source_type    => ORDS.source_type_plsql,
    p_source         => q'[
      DECLARE
        l_payload CLOB;
      BEGIN
        l_payload := psych_first_responder_api.build_handoff(:session_id);
        :body := l_payload;
      END;
    ]'
  );

  COMMIT;
END;

Guardrail matrix

                
Make the policy visible instead of burying it in prose.

                

                  

                    
                      

                        Signal captured
                        Service response
                        Runtime outcome
                      

                    
                    
                      

                        Active self-harm intent, current plan, or inability to stay safe
                        Stop normal intake, surface the safety message, and flag the handoff as immediate escalation.
                        Route becomes IMMEDIATE_SAFETY_ESCALATION.
                      

                      

                        No self-harm intent, but severe panic, rapid worsening, or multiple nights without sleep
                        Continue only enough to capture supports, then mark the case for urgent psychiatric review.
                        Route becomes URGENT_PSYCHIATRY_REVIEW.
                      

                      

                        Stable pre-visit intake with no acute safety flags
                        Complete the guided flow, capture the rapid assessment, and assemble the standard handoff.
                        Route becomes ROUTINE_PSYCHIATRY_INTAKE.
                      

                      

                        Model tries to diagnose, prescribe, or invent citations
                        Suppress that answer, return policy-safe guidance, and use approved chunks only.
                        Guardrail event is logged for review.

Prompt contract

                    
System instruction the workshop can explain

                    
Keep the prompt contract concrete so the learner sees what the assistant must do and what it must never do.

                  

                  
                

                
system_role: psychiatry first responder for pre-visit intake
goals:
  - collect presenting concern, timeline, mood, anxiety, sleep, function, safety, current care, and supports
  - use DSM-5-informed symptom domains without diagnosing
  - add a CBT-style thought and behavior note for the clinician handoff
  - retrieve approved clinic guidance only when needed
must_do:
  - ask safety questions directly and early
  - stop normal intake when immediate safety escalation triggers
  - return a concise clinician handoff and PDF payload
must_not:
  - diagnose
  - prescribe or adjust medication
  - invent policy or citations
  - continue a normal script when the patient is unsafe

              


              

                

                  

                    
APEX wiring

                    
Call pattern for handoff generation and PDF export

                    
Short enough to teach, realistic enough to feel like a workshop, and directly connected to the report preview the clinician will open.

                  

                  
                

                
apex.server.process(
  "GENERATE_PSYCH_HANDOFF",
  {
    x01: $v("P10_SESSION_ID"),
    x02: $v("P10_ASSESSMENT_TOTAL")
  },
  {
    dataType: "json",
    success: function (payload) {
      apex.item("P10_ROUTE_CODE").setValue(payload.route_code);
      apex.item("P10_REPORT_SUMMARY").setValue(payload.report_summary);
      apex.item("P10_PDF_STATUS").setValue(payload.pdf_status);
      apex.region("clinicianHandoffPreview").refresh();
    }
  }
);
